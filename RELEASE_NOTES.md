# v1.2.0

## 本次更新

- 唯一入睡触发源仍为 `SleepModeManager.t(SleepModelSettings)` 中的 `isStartNow=true`。
- 将“启用入睡闹钟计划”开关拆分为独立卡片。
- 开关切换后立即写入并同步到健康 `:transport` 进程，无需点击“保存计划”。
- “保存计划”现在只保存计划睡眠时长和闹钟名称。
- 成功创建一次性闹钟后，立即自动关闭模块自己的计划开关。
- 再次使用时，只需重新打开开关，状态即时生效。

## 锁屏流程

- 系统框架放行整个 `com.coloros.alarmclock` 时钟包。
- 健康进程先拉起时钟主程序。
- 时钟取得前台资格后，由时钟自身 UID 创建一次性闹钟。
- 创建完成后自动关闭临时时钟页面。

## 兼容性

- OPPO 健康：`6.4.6_cb99e90_260626`
- OPPO 时钟：`16.19.0`
- Android：12 及以上
- LSPosed 作用域：`OPPO 健康`、`时钟`、`系统框架`
